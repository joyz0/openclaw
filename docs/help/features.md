# OpenClaw 功能特性说明

## 1. 核心功能

### 1.1 多渠道消息集成
- **支持的渠道**：WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage（通过BlueBubbles）、Microsoft Teams、Matrix、Zalo、Zalo Personal、WebChat
- **消息路由**：支持按渠道、账户和对等方路由到隔离的代理
- **群组消息**：支持群组激活模式、提及门控和回复标签
- **安全默认值**：默认采用DM配对策略，未知发送者需要通过配对码验证

### 1.2 本地优先网关
- **WebSocket控制平面**：单个WebSocket服务器用于会话、状态、配置、定时任务和Webhook
- **Tailscale集成**：支持Tailscale Serve（仅tailnet）或Funnel（公共）
- **远程访问**：通过SSH隧道或Tailscale实现远程网关控制
- **安全认证**：支持密码认证和Tailscale身份验证

### 1.3 代理系统
- **Pi代理运行时**：RPC模式下的代理运行时，支持工具流和块流
- **会话模型**：主会话用于直接聊天，群组隔离，激活模式，队列模式，回复回传
- **多代理路由**：将入站渠道/账户/对等方路由到隔离的代理（工作区+每个代理会话）
- **会话工具**：支持会话列表、历史记录和跨会话消息发送

### 1.4 工具和自动化
- **浏览器控制**：专用的openclaw Chrome/Chromium，支持快照、操作和上传
- **Canvas**：代理驱动的可视化工作区，支持A2UI推送/重置、评估和快照
- **节点**：摄像头快照/剪辑、屏幕录制、位置获取、通知
- **定时任务和唤醒**：支持cron任务和定时唤醒
- **Webhook**：支持外部触发器通过webhook表面
- **Gmail Pub/Sub**：支持Gmail邮件触发

### 1.5 语音和Talk模式
- **Voice Wake**：macOS/iOS/Android上的始终开启语音唤醒
- **Talk模式**：持续对话模式，支持语音输入和输出
- **ElevenLabs集成**：高质量语音合成

### 1.6 技能平台
- **ClawHub**：最小化技能注册表，代理可以自动搜索和拉取新技能
- **捆绑技能**：核心捆绑的基础技能
- **托管和工作区技能**：支持托管和用户自定义工作区技能
- **安装门控**：技能安装需要用户授权

## 2. 应用和节点

### 2.1 macOS应用
- **菜单栏控制**：网关和健康状态的菜单栏控制
- **Voice Wake + 按键通话**：语音唤醒和按键通话覆盖
- **WebChat + 调试工具**：内置WebChat和调试工具
- **远程网关控制**：通过SSH远程控制网关

### 2.2 iOS节点
- **Bridge配对**：通过Bridge作为节点配对
- **语音触发器转发**：语音触发转发到网关
- **Canvas表面**：iOS上的Canvas界面
- **相机和屏幕录制**：支持相机快照和屏幕录制

### 2.3 Android节点
- **Bridge配对**：与iOS相同的Bridge配对流程
- **Canvas、相机和屏幕捕获**：支持Canvas、相机和屏幕捕获命令
- **可选SMS**：可选的SMS支持

### 2.4 macOS节点模式
- **系统运行/通知**：支持系统命令运行和通知
- **Canvas/相机暴露**：暴露Canvas和相机功能

## 3. 安全和隐私

### 3.1 安全默认值
- **DM访问控制**：默认采用DM配对策略，未知发送者需要配对码
- **群组安全**：可以设置`agents.defaults.sandbox.mode: "non-main"`在Docker沙箱中运行非主会话
- **沙箱默认值**：允许列表包括bash、process、read、write、edit、sessions_*，拒绝列表包括browser、canvas、nodes、cron、discord、gateway

### 3.2 权限管理
- **macOS权限**：通过Gateway协议管理macOS权限，包括屏幕录制、通知等
- **提升访问**：使用`/elevated on|off`切换每个会话的提升访问权限
- **权限映射**：节点通过Gateway WebSocket广告其功能和权限映射

### 3.3 安全模型
- **主会话**：工具在主机上运行，代理具有完全访问权限
- **非主会话**：可以在Docker沙箱中运行，限制访问权限
- **安全指南**：详细的安全指南和最佳实践

## 4. 配置和管理

### 4.1 配置系统
- **最小配置**：只需设置模型和默认值
- **完整配置**：支持详细的配置选项，包括渠道、安全、工具等

### 4.2 命令行界面
- **网关管理**：启动、停止、配置网关
- **代理管理**：发送消息、重置会话、设置思考级别
- **渠道管理**：登录、配置、测试渠道
- **工具管理**：浏览器、Canvas、节点等工具的管理
- **系统管理**：健康检查、日志查看、系统状态

### 4.3 监控和日志
- **健康检查**：网关健康状态检查
- **日志系统**：详细的日志记录和分析
- **状态监控**：会话状态、模型使用情况、成本统计

## 5. 扩展和插件

### 5.1 插件系统
- **插件API**：广泛的插件API
- **npm包分发**：首选的插件分发路径
- **本地扩展加载**：支持本地扩展加载用于开发
- **社区插件**：社区插件列表和贡献指南

### 5.2 内存插件

### 5.3 MCP支持
- **mcporter**：通过mcporter支持MCP
- **灵活集成**：保持MCP集成灵活且与核心运行时解耦
- **桥接模型**：偏好桥接模型而非在核心中构建一等MCP运行时

## 6. 部署选项

### 6.1 本地部署
- **Node.js运行时**：需要Node ≥22
- **包管理器**：支持npm、pnpm或bun
- **向导安装**：`openclaw onboard --install-daemon`
- **后台服务**：安装Gateway守护进程（launchd/systemd用户服务）

### 6.2 Docker部署
- **Docker支持**：基于Docker的安装
- **沙箱**：用于非主会话的Docker沙箱
- **容器化**：完整的容器化支持

### 6.3 远程部署
- **Linux实例**：可以在小型Linux实例上运行Gateway
- **客户端连接**：通过Tailscale Serve/Funnel或SSH隧道连接客户端
- **设备节点**：可以配对设备节点（macOS/iOS/Android）以在需要时执行设备本地操作

## 7. 开发和贡献

### 7.1 从源代码构建
- **首选pnpm**：从源代码构建时首选pnpm
- **Bun可选**：可选使用Bun直接运行TypeScript
- **开发循环**：支持TypeScript更改的自动重载

### 7.2 贡献规则
- **PR范围**：一个PR = 一个问题/主题，不要捆绑多个不相关的修复/功能
- **代码大小**：超过~5,000行更改的PR仅在特殊情况下进行审核
- **PR批量**：不要一次打开大量小PR，每个PR都有审核成本
- **小修复分组**：对于非常小的相关修复，鼓励分组到一个集中的PR中

### 7.3 不合并的内容（暂时）
- **新核心技能**：当它们可以在ClawHub上存在时
- **完整文档翻译**：推迟；计划稍后进行AI生成的翻译
- **商业服务集成**：不符合模型提供商类别的商业服务集成
- **包装渠道**：围绕已支持渠道的包装渠道，没有明确的功能或安全差距
- **核心中的一等MCP运行时**：当mcporter已经提供集成路径时
- **代理层次结构框架**：作为默认架构的管理器（manager-of-managers / 嵌套规划树）
- **重型编排层**：复制现有代理和工具基础架构的重型编排层

## 8. 技术栈

### 8.1 主要语言
- **TypeScript**：主要编排系统语言，选择TypeScript是为了保持OpenClaw默认可破解

### 8.2 核心依赖
- **Node.js**：运行时环境（≥22）
- **WebSocket**：网关控制平面
- **Docker**：沙箱和容器化
- **Tailscale**：远程访问和网络

### 8.3 渠道集成
- **WhatsApp**：Baileys
- **Telegram**：grammY
- **Slack**：Bolt
- **Discord**：discord.js
- **Google Chat**：Chat API
- **Signal**：signal-cli
- **iMessage**：BlueBubbles（推荐）或legacy imsg

## 9. 快速开始

### 9.1 安装
```bash
npm install -g openclaw@latest
# 或: pnpm add -g openclaw@latest

openclaw onboard --install-daemon
```

### 9.2 运行网关
```bash
openclaw gateway --port 18789 --verbose
```

### 9.3 发送消息
```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

### 9.4 与代理对话
```bash
openclaw agent --message "Ship checklist" --thinking high
```

## 10. 总结

OpenClaw是一个强大的个人AI助手，它在您自己的设备上运行，通过您已经使用的渠道与您交互，并遵循您的规则。它提供了广泛的功能，包括多渠道集成、本地优先网关、代理系统、工具和自动化、语音和Talk模式、技能平台、应用和节点、安全和隐私、配置和管理、扩展和插件、部署选项等。

OpenClaw的目标是成为一个易于使用、支持广泛平台、尊重隐私和安全的个人助手。它目前处于快速迭代阶段，重点关注安全和安全默认值、错误修复和稳定性、设置可靠性和首次运行用户体验。

通过其模块化设计和扩展能力，OpenClaw为用户提供了一个灵活、强大的AI助手平台，可以根据个人需求进行定制和扩展。
