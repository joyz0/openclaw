# OpenClaw 扩展自定义大模型指南

OpenClaw 支持通过配置文件或命令行扩展自定义大模型，包括千问模型。以下是详细的扩展方法：

## 一、通过配置文件添加自定义模型

### 1. 了解配置结构

OpenClaw 使用 `~/.openclaw/config.json` 文件存储模型配置，核心结构包括：

```json
{
  "models": {
    "providers": {
      "<providerId>": {
        "baseUrl": "https://api.example.com",
        "apiKey": "YOUR_API_KEY",
        "api": "openai-completions",
        "models": [
          {
            "id": "model-id",
            "name": "Model Name",
            "reasoning": true,
            "input": ["text", "image"],
            "cost": {
              "input": 0.001,
              "output": 0.002,
              "cacheRead": 0,
              "cacheWrite": 0
            },
            "contextWindow": 128000,
            "maxTokens": 4096
          }
        ]
      }
    }
  }
}
```

### 2. 添加千问模型配置

## 二、通过命令行添加自定义模型

OpenClaw 提供了命令行工具来添加自定义模型：

### 1. 使用 `onboard-custom` 命令

```bash
# 添加千问模型
npx openclaw onboard-custom --custom-base-url https://ark.cn-beijing.volces.com/api/v3 --custom-model-id qwen-plus --custom-provider-id qwen
```

### 2. 交互式配置

运行以下命令进入交互式配置：

```bash
npx openclaw onboard-custom
```

然后按照提示输入：

- API Base URL：`https://ark.cn-beijing.volces.com/api/v3`
- API Key：你的千问 API Key
- 模型 ID：`qwen-plus` 或其他千问模型 ID
- Provider ID：`qwen`（或其他你喜欢的名称）

## 三、通过认证配置添加模型

### 1. 千问门户认证

OpenClaw 已经内置了千问门户的认证支持：

```bash
npx openclaw auth
```

然后选择 "Qwen Portal" 进行认证配置。

### 2. 自定义认证配置

如果需要为新的模型提供商添加认证支持，可以：

1. 在 `src/commands/auth-choice.apply.ts` 中添加新的认证处理函数
2. 将该函数添加到 `handlers` 数组中

## 四、配置说明

### 1. 关键配置字段

- **baseUrl**：模型 API 的基础 URL
- **apiKey**：API 密钥
- **api**：API 类型（如 `openai-completions`、`anthropic-messages` 等）
- **models**：模型列表，每个模型包含：
  - **id**：模型 ID
  - **name**：模型名称
  - **reasoning**：是否支持推理
  - **input**：支持的输入类型（`text`、`image` 等）
  - **cost**：成本配置
  - **contextWindow**：上下文窗口大小
  - **maxTokens**：最大令牌数

### 2. 支持的 API 类型

OpenClaw 支持以下 API 类型：

- `openai-completions`
- `openai-responses`
- `anthropic-messages`
- `google-generative-ai`
- `github-copilot`
- `bedrock-converse-stream`
- `ollama`

## 五、使用自定义模型

添加自定义模型后，可以通过以下方式使用：

1. **命令行指定**：

   ```bash
   npx openclaw chat --model qwen/qwen-plus
   ```

2. **会话中切换**：

   ```
   /model qwen/qwen-plus
   ```

3. **设置默认模型**：在配置文件中设置默认模型

## 总结

OpenClaw 提供了灵活的机制来扩展自定义大模型，包括：

1. **配置文件方式**：直接在 `config.json` 中添加模型配置
2. **命令行方式**：使用 `onboard-custom` 命令交互式添加
3. **认证配置**：通过 `auth` 命令配置模型认证
4. **内置支持**：已经内置了千问门户的认证支持

通过这些方法，你可以轻松地在 OpenClaw 中添加千问等自定义大模型，享受与内置模型相同的功能和体验。
