# OpenClaw 配置说明报告

本文档详细列出了 OpenClaw 项目支持的所有配置项，包括配置路径、类型、默认值和说明。

## 1. 配置结构概览

OpenClaw 采用分层配置结构，主要配置项包括：

- `meta` - 配置元数据
- `env` - 环境配置
- `wizard` - 向导配置
- `diagnostics` - 诊断配置
- `logging` - 日志配置
- `update` - 更新配置
- `browser` - 浏览器配置
- `ui` - 用户界面配置
- `auth` - 认证配置
- `models` - 模型配置
- `nodeHost` - 节点主机配置
- `agents` - 代理配置
- `tools` - 工具配置
- `bindings` - 代理绑定
- `broadcast` - 广播配置
- `audio` - 音频配置
- `media` - 媒体配置
- `messages` - 消息配置
- `commands` - 命令配置
- `approvals` - 审批配置
- `session` - 会话配置
- `cron` - 定时任务配置
- `hooks` - 钩子配置
- `web` - Web 配置
- `channels` - 通道配置
- `discovery` - 发现配置
- `canvasHost` - Canvas 主机配置
- `talk` - 语音配置
- `gateway` - 网关配置
- `memory` - 内存配置
- `skills` - 技能配置
- `plugins` - 插件配置

## 2. 详细配置项

### 2.1 meta

配置元数据，记录配置的修改信息。

| 配置路径 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `meta.lastTouchedVersion` | string | - | 最后修改此配置的 OpenClaw 版本 |
| `meta.lastTouchedAt` | string | - | 此配置最后被修改的 ISO 时间戳 |

### 2.2 env

环境配置，包括环境变量和 shell 环境。

| 配置路径 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `env.shellEnv.enabled` | boolean | - | 是否启用从登录 shell 环境导入缺失的密钥 |
| `env.shellEnv.timeoutMs` | number | 15000 | 登录 shell 执行的超时时间（毫秒） |
| `env.vars` | Record<string, string> | - | 内联环境变量，当进程环境中不存在时应用 |
| `env.*` | string | - | 直接在 env 下定义的环境变量（字符串值） |

### 2.3 wizard

向导配置，记录向导的运行信息。

| 配置路径 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `wizard.lastRunAt` | string | - | 向导最后运行的时间 |
| `wizard.lastRunVersion` | string | - | 向导最后运行的版本 |
| `wizard.lastRunCommit` | string | - | 向导最后运行的提交 |
| `wizard.lastRunCommand` | string | - | 向导最后运行的命令 |
| `wizard.lastRunMode` | "local" \| "remote" | - | 向导最后运行的模式 |

## 3. 配置文件格式

OpenClaw 使用 JSON5 格式的配置文件，支持注释和尾随逗号。配置文件默认存储在用户目录中。

## 4. 配置优先级

OpenClaw 配置优先级从高到低为：

1. 命令行参数
2. 环境变量
3. 配置文件
4. 默认值

## 5. 配置示例

以下是一个基本的配置示例：

```json5
{
  "meta": {
    "lastTouchedVersion": "2026.2.23",
    "lastTouchedAt": "2026-02-24T00:00:00Z"
  },
  "logging": {
    "level": "info",
    "consoleLevel": "info",
    "consoleStyle": "pretty"
  },
  "update": {
    "channel": "stable",
    "checkOnStart": true
  },
  "browser": {
    "enabled": true,
    "headless": false
  },
  "agents": {
    "defaults": {
      "model": "gpt",
      "maxConcurrent": 4
    }
  },
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "auto",
    "controlUi": {
      "enabled": true
    }
  }
}
```

## 6. 配置管理命令

OpenClaw 提供了以下命令来管理配置：

- `openclaw config get <path>` - 获取配置值
- `openclaw config set <path> <value>` - 设置配置值
- `openclaw config unset <path>` - 取消设置配置值
- `openclaw configure` - 交互式配置向导

## 7. 注意事项

- 敏感配置项（如 API 密钥）会在日志中脱敏
- 配置文件中的 `$schema` 字段用于编辑器工具，但不会在控制 UI 中显示
- 插件和通道可以通过元数据扩展配置模式和 UI 提示
- 某些配置项需要重启网关才能生效

## 8. 总结

本文档提供了 OpenClaw 项目所有配置项的详细说明，可作为配置 OpenClaw 的参考指南。随着项目的发展，配置项可能会发生变化，请参考最新的源代码和文档获取最准确的信息。
